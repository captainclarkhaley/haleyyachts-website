## What this file is (and is not)

- Candidate Haley-website features to consider before adding. Candidates, not commitments.
- Distinct from committed / in-flight work tracked in:

  - `tasks/terry.md` - Terry's engineering list.
  - `tasks/patrick.md` - Patrick's marketing / content list.
- "Must Have" items are NOT parked here; they go straight onto the task list.

## The flow

```
idea  ->  Wish List (consider)  ->  if approved, promote to the task list to build
```

When an item is approved to build, move it onto the owner task file (`tasks/terry.md` or `tasks/patrick.md`) and regenerate `TASK-LIST.md`. Leave a short note here that it was promoted, with the date.
